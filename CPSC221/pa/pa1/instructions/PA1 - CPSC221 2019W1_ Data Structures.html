The Assignment, Part 2: Coding

Problem Specification

A Linked List is a dynamic linear structure designed to hold any type of data. In this exercise, we develop and use a coordinated set of linked lists to manipulate blocks of pixels from an image.

We have broken the image below into 15 Blocks.

Each Block is placed into a Node of a Grid, as shown here:

Each row and each column of the Grid forms a circular doubly-linked list. The Grid can be rearranged, and the image reassembled to create fascinating visual results.

We have provided a starting point for achieving this functionality. It is your task to complete and expand on our implementation.

Specifications for each function you write are contained in the given code. The list of functions here should serve as a checklist for completing the exercise.

In block.cpp

  • int width() const; Returns the width of the block.
  • int height() const; Returns the height of the block.
  • void Block(PNG & im, int x, int y, int width, int height); Makes a block from the rectangle of width by height pixels in im whose upper-left corner is at position (x,y).
  • void render(PNG & im, int x, int y ) const; Draws the block at position (x,y) in im.
  • void greyscale(); Changes the saturation of every pixel in the block to 0, which removes the color, leaving grey.

In grid.cpp

  • void clear(); Helper function for destructor and assignment operator.
  • void copy(const Grid & other); Helper function for copy constructor and assignment operator.
  • ~Grid(); Destructor.
  • rotateR(int r, int count); Rotate row r by count positions.
  • rotateC(int c, int count); Rotate column c by count positions.

In grid.h

  • PUZZLE_SOLUTION The sequence of rotateR and rotateC commands to produce the right image from the left image shown below.

Implementation Constraints and Advice

We will be grading your work on functionality, efficiency, and memory use. All Grid functionality, aside from the copy function, can be achieved by moving existing nodes, rather than by allocating new ones and/or making copies. If you are tempted to use the new function when you are manipulating the Grid, ask yourself if you can achieve your goal by reassigning pointers, instead.

The Grid structure is a two-dimensional circular doubly-linked list of Nodes with two vectors of pointers to access the first Node in each row and column. Each Node contains four pointers up, down, left, and right. Repeatedly following the right pointer from Node to Node will visit all the Nodes in a row and eventually return to the starting Node. The same is true of following left pointers; and following up or down pointers visits all the Nodes in a column.

Finally, you’ll note that we are asking you to implement special memory management functions for the Grid class. You should ask yourself why we have not made similar requests for the Block class. Why doesn’t it need a destructor, copy constructor, or assignment operator?

Handing in your code

To facilitate anonymous grading, do not include any personally-identifiable information (like your name, your University ID number, or your cwl) in any of your source files. Instead, before you hand in this assignment, create a file called partners.txt that contains only the CSIDs of people in your collaboration partnership (if one exists), one per line. If you worked alone, include only your own CSID in this file. A computer will process this information using a simple algorithm, so do not include anything else in the file. (If we must manually correct your submission, you may lose points.) As always, if you’re working in a group, only one group member needs to hand in the assignment.

The following files are used to grade PA1:

  • block.cpp
  • grid.cpp
  • grid.h
  • partners.txt

All other files will not be used for grading.

To submit your work for autograding, simply navigate to “PA1 – Block Grid” on Gradescope and drag and drop the files listed above.
